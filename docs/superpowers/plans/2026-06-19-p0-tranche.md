# P0 Tranche Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement custom exception hierarchy, fix LSP violations in abstract method signatures, and add a unittest-based test suite covering all core modules.

**Architecture:** New `exceptions.py` module defines a typed hierarchy rooted at `XPWebAPIError`. Existing exception classes in `beacon.py` and `udp.py` become subclasses for backward compatibility. Abstract methods in `api.py` are cleaned up to match their override contracts. Tests use `unittest.mock` to patch `httpx`, `simple_websocket`, and `socket` — no live X-Plane instance required.

**Tech Stack:** Python 3.12+, `unittest` (stdlib), `unittest.mock`, `httpx`, `simple-websocket`

## Global Constraints

- Python >= 3.12
- No pytest. Use `unittest` (stdlib) exclusively.
- Test command: `uv run python -m unittest discover`
- Lint: `uv run ruff check xpwebapi/ tests/`
- Format: `uv run ruff format xpwebapi/ tests/`
- Type check: `uv run ty check` (if available, non-blocking)
- Old exception names (`XPlaneNoBeacon`, `XPlaneVersionNotSupported`, `XPlaneTimeout`) must remain importable from `xpwebapi`
- Do not modify `examples/` directory

        packet += struct.pack("<if", idx, value)
    return packet

        d = Dataref(path="sim/test/val", api=api)
        d._cached_meta = meta
